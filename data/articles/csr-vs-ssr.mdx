## Client Side Rendering

![enter image description here](https://raw.githubusercontent.com/malahngoding/stand/Badge/public/images/csr.png)
**Client-Side Rendering (CSR)** adalah mode rendering di mana permintaan untuk mengakses halaman diproses oleh browser, hanya menggunakan file JavaScript untuk menghasilkan konten HTML.

Informasi perintah akses dikirim ke server, yang kembali dengan HTML sederhana yang berisi informasi JavaScript yang akan memuat seluruh situs.

Ini adalah metode rendering yang jauh lebih modern karena tidak ada penerimaan seluruh halaman HTML. Dari segi teknis, mode pemuatan ini membuat proses lebih cepat, membutuhkan lebih sedikit dari server dan browser.

Metode CSR selalu menjadi pilihan, tetapi selalu penting untuk mengevaluasi pro dan kontra, yang masih akan kami lakukan dalam konten ini.

## Server Side Rendering

![enter image description here](https://raw.githubusercontent.com/malahngoding/stand/Badge/public/images/ssr.png)
**Server-Side Rendering (SSR)** adalah metode yang dianggap lebih tua dan lebih berat daripada CSR. Di sini, ketika pengguna mengakses situs web, informasi permintaan ini dikirim langsung ke server, yang merespons dengan file dengan kode HTML lengkap. Dari konten ini, situs web akan dimuat dengan benar secara penuh.

Dalam metode ini, server membuat halaman dari kode HTML dan data akses pengguna yang mengakses situs. SSR telah menghabiskan banyak waktu menjadi cara paling umum untuk merender halaman, tetapi analisis dan diskusi tentang metode mana yang terbaik mempertanyakan penggunaan teknik ini.

### **Client Server Rendering vs Server Side Rendering**

**Kelebihan CSR:**

1.  Request cukup 1 halaman diawal, Selanjutnya hanya manipulasi DOM.
2.  Fleksible, mudah untuk dioptimasi.
3.  Biaya server dan pemeliharaan murah.
4.  Interaktif

**Kekurangan CSR:**

1.  Search Engine Optimization, Search engine susah untuk merambah web dan mengindeks halaman. Butuh langkah ekstra.
2.  Social Media Optimization, tidak ada tampilan preview halaman web berdasarkan open graph saat di share. Butuh langkah ekstra.
3.  Ukuran file JavaScript yang diunduh dan dieksekusi berpengaruh pada performa.

**Kelebihan SSR:**

1.  Search Engine Optimization (SEO), Search engine mudah untuk merambah web dan mengindeks halaman.
2.  Social Media Optimization, Tampilan preview halaman web berdasarkan open graph saat di share.
3.  Performa yang cepat saat halaman sudah sepenuhnya tampil (loaded) karena render sudah dilakukan di sisi server tanpa ada loading spinner.

**Kekurangan SSR:**

1.  Request 1 halaman penuh di setiap navigasinya.
2.  Tidak fleksibel, Saat aplikasi menjadi besar butuh banyak optimasi disisi server.
3.  Biaya server dan pemeliharaan lumayan besar.
4.  Tidak interaktif.
5.  Beban di sisi server

### **Jadi kapan kita memilih CSR atau SSR?**

**Gunakan CSR saat:**

- Ketika SEO bukan prioritas
- Jika web memiliki banyak interaksi. contoh chat.
- Jika Kalian sedang membangun aplikasi web, contohnya admin dashboard.

**Gunakan SSR saat:**

- SEO adalah prioritas. seperti web blog atau berita.
- Tidak butuh banyak interaksi.
- Jika kalian membutuhkan tampilan web secara keseluruhan diawal lebih cepat, tanpa ada loading spinner.

Analisis Client Side Rendeing vs. Server Side Rendering sangat berharga dan harus menjadi masalah berharga bagi desainer web. Yang mendesain website berkomitmen untuk selalu memikirkan yang terbaik bagi pengguna, terutama dalam hal performa.
